# Accomi

> 一盏灯，一段共时——「我在，你也在。」

---

## 1. 一句话

**Accomi** 是一款双人弱连接 App：用灵动岛同步彼此的在线状态，安静记录你们「同时在场」的时光。App 只提供几种基础信号原语；至于这些信号意味着什么——摩尔斯电码、暗号、晚安仪式——由你们自己定义。

---

## 2. 设计哲学：弱连接 · 轻交互 · 可延伸

### 弱连接

| 意味着 | 不意味着 |
|--------|----------|
| 在线是一种邀请，离线是自然状态 | 「你怎么不在线」的隐性压力 |
| 状态可见，但不需要回应 | 已读回执、回敲、确认收到 |
| 有期限的会话（8h）自动结束 | 24 小时互相监视 |
| 时间记录是「回忆」，不是 KPI | 连续打卡、成就、排行榜 |

### 轻交互

| 层级 | 交互 | 频率 |
|------|------|------|
| **零交互** | 灵动岛瞥见对方在线 | 日常主力 |
| **一次交互** | 打开 App / 长按熄灯 | 想在场或离开时 |
| **原语交互** | 点对方灯叩 · 长按对方灯长叩 | 偶尔 |
| **浏览交互** | 上滑看共时 / 信号记录 | 偶尔回顾 |

### 可延伸

App 是**传输层**，不是**语义层**。

- 系统忠实传递信号的形态与时序，不解读、不解码、不提示含义
- 用户可在原语之上自行约定玩法——摩尔斯电码、节拍暗号、固定仪式
- 不玩的用户，单点一下照样是「轻叩」，零学习成本

## 3. 产品决策（已确认）

| 项 | 决定 |
|----|------|
| 共时记录保留 | **永久**，双方本地 + 云端同步 |
| 离线主屏 | **显示**「上次同在 · 时间 · 时长」 |
| 轻叩 | **纳入 MVP**，点对方灯触发 |
| 灵动岛光点 | **单一光点**——亮 = 对方在线，暗 = 离开 |
| 入口结构 | **打开 App → 主页**；配对、记录、叩击均在此 |
| 后台行为 | **已配对时划出 App → 灵动岛 / Live Activity 接管** |
| 主动离线 | **长按自己灯 → 熄灯**，替代杀进程作为主要离开方式 |
| 交互原语 | **短叩 + 长叩**，时序保真，支持用户延伸玩法 |
| 信号记录 | **永久保存**接收到的叩击序列，支持回放；无已读回执 |

## 4. App 生命周期

```
启动 App
  └─→ 主页
        ├─ 未配对 → 配对区（邀请码 / 扫码）
        └─ 已配对 → 双灯 · 共时 · 记录 · 叩击

划出 App（已配对）
  └─→ 维持在线，灵动岛 / Live Activity 接管
        ├─ 灵动岛：单一光点（对方在线 = 亮）
        └─ 锁屏：双灯 + 本次共时

长按自己灯 / 主动熄灯
  └─→ 灯 3s 渐灭，离线，共时写入记录

杀进程
  └─→ 同主动熄灯，灯灭离线
```

| 状态 | 你 | 对方看到 |
|------|----|----------|
| App 在前台 | 在线，主页可操作 | 灵动岛光点亮 |
| App 在后台 | 仍在线，Live Activity 运行 | 灵动岛光点亮 |
| 主动熄灯 / 杀进程 | 离线，灯 3s 渐灭 | 灵动岛光点渐灭 |

**首次配对引导**（一次性，之后不再出现）：

> 「划出 App，灯会继续亮着。想离开时，长按你的灯。」

## 5. 交互原语体系

Accomi 的所有主动互动，由 **3 个原子原语** 组合而成。App 只负责传递，不负责翻译。

### 5.0 原语一览

| 原语 | 操作 | 传递内容 | 接收反馈 |
|------|------|----------|----------|
| **灯亮** | 打开 App / 划出后台 | 在线状态 | 对方灵动岛光点亮 |
| **灯灭** | 长按自己灯 / 杀进程 | 离线状态 | 对方光点 3s 渐灭 |
| **短叩** | 点对方灯 | 一次短脉冲 | 灯闪 + soft Haptic |
| **长叩** | 长按对方灯 ~0.6s | 一次长脉冲 | 灯长闪 + medium Haptic |

**时序保真**：连续叩击时，系统按**真实间隔**逐次传递。间隔长短本身成为信息载体——这是摩尔斯电码、节拍暗号能成立的基础。

**频率限制**：单方向 30 秒内最多触发 8 次叩击（含长短），防止焦虑型刷屏。触顶时发送方自己的灯微暗一下，无文字提示。

### 5.1 用户延伸玩法（App 不参与）

App 不提供摩尔斯键盘、暗号词典或玩法教程。以下为**用户自发约定**的示例：

| 玩法 | 原语组合 | 说明 |
|------|----------|------|
| **摩尔斯电码** | 短叩 = · ，长叩 = — | 如 `— — · ·` = 「M」；双方事先约定即可 |
| **固定仪式** | 短叩 × 3 | 「我困了」「到家了」—— 含义由关系定义 |
| **节拍传情** | 短叩-短叩-长叩-短叩 | 类似心跳节奏，无需文字 |
| **叩而不回** | 任意序列 | 发完即走，接收方无需回应——弱连接不变 |

**设计边界**：

| App 做 | App 不做 |
|--------|----------|
| 保真传递短/长叩与时序 | 摩尔斯编解码 |
| 接收端灯闪 + Haptic | 「消息已读 / 请回叩」 |
| **离线留存 + 信号记录 + 回放** | 推送「对方叩了你」 |
| 首次配对后可选提示「叩击可以组成节奏」 | 玩法攻略、成就、解锁 |

### 5.2 信号留存 · 异步送达

接收方不可能时时盯着屏幕。叩击因此有**两条送达路径**：

```
发送方叩击
  ├─ 接收方 App 在前台  →  实时灯闪 + Haptic + 写入信号记录
  └─ 接收方未在看        →  仅写入信号记录，待打开 App 后感知
```

**分段规则**：连续叩击间隔 < 2.5s 视为同一段信号；超过 2.5s 静默则切段，各段独立保存。一段即一个「叩击序列」，可对应一个摩尔斯词或一次仪式。

**打开 App 时（有未读信号）**：

1. 对方灯旁出现极淡未读光环（非红色角标）
2. 你的灯自动回放**最新一段**未读序列（一次）
3. 回放结束，该段标记为已看——**仅本地，不通知发送方**

**不是 IM**：无气泡、无已读回执、无「请回复」；信号记录是**留声机**，不是聊天框。

## 6. 核心功能

### 6.1 灯 · 在线状态

**主页视觉**：纯黑背景，两盏暖色呼吸发光圆，安静并排。

- **同亮呼吸同步**（V0.2）：双方同时在场时，两灯呼吸节律一致
- **离线渐灭**：对方离开时，其灯 3s 渐灭，非瞬灭
- **灵动岛**：单一光点，亮 = 对方在线，暗 = 离开
- **锁屏 Live Activity**：双灯 + 本次共时
- **8 小时会话**：到期自然熄灭，共时写入，不推送

### 6.2 时 · 共时记录

只记录**双方同时在线**的重叠时段。

**主屏信息优先级**（互斥显示）：

```
1. 双方同亮  →  「共时 47 分钟」
2. 否则       →  「上次同在 昨天 23:14 · 1h 08m」
3. 无历史     →  仅双灯，无文字
```

**中间态**（仅一方在线）：

| 状态 | 你的灯 | 对方灯 | 底部文案 |
|------|--------|--------|----------|
| 仅你在 | 亮 | 暗 | 上次同在（若有） |
| 仅对方在 | 暗 | 亮 | 上次同在（若有） |
| 双方同亮 | 亮 | 亮 | 共时 N 分钟 |

**共时列表**（上滑）：永久保留，按年/月分组折叠，自然周汇总。

### 6.3 叩 · 主动信号

**点对方灯 → 短叩** · **长按对方灯 → 长叩**

- 接收方在前台：你的灯闪 + Haptic，**无回叩按钮**
- 接收方不在看：信号写入记录，打开 App 后回放
- 发送方：对方灯短暂闪一下表「已送达」（不代表对方已看）

### 6.4 信号 · 叩击记录

与共时记录并列，永久保存**收到**的叩击序列。

**记录格式**（不解码，只存形态）：

```
今天 14:32        · — · · —
昨天 23:05        · · ·
6/25  00:18       — · — — · —
```

- **·** = 短叩 · **—** = 长叩 · 间隔信息内嵌于序列
- 点击某条 → 在你的灯上回放该序列
- 按年/月分组，永久保留
- 发送方**看不到**对方是否已看

**与共时的区别**：

| | 共时 | 信号 |
|--|------|------|
| 内容 | 双方同亮的时间段 | 对方发来的叩击序列 |
| 触发 | 自动 | 对方主动 |
| 形态 | `1h 08m` | `· — · ·` |
| 情感 | 「我们在一起待了多久」 | 「ta 敲了什么给我」 |

## 10. MVP（V0.1）

### 做

1. **主页** — 配对 + 双灯 + 共时 + 记录
2. **配对** — 邀请码绑定
3. **在线状态** — 前台/后台在线，长按熄灯 / 杀进程离线
4. **灵动岛** — 单一光点
5. **锁屏 Live Activity** — 双灯 + 本次共时
6. **共时** — 同亮累加，永久列表，主屏优先级逻辑
7. **离线主屏** — 上次同在
8. **短叩 + 长叩** — 点/长按对方灯，时序保真传递
9. **接收反馈** — 前台实时闪振；后台写入记录
10. **信号记录** — 永久保存、分段、未读回放、点击回放
11. **首次引导** — 划出在线 + 长按熄灯 + 可叩提示（各一次）

### 不做

- 摩尔斯编解码、暗号词典、玩法教程
- 回叩按钮、**发送方已读回执**、消息气泡
- 推送「对方叩了你」
- 白噪音 / 推送通知
- streak、成就、单方在线追踪
- 灵动岛扩展态 / 多人

### 验收标准

> 配对后划出 App，灵动岛光点亮。
> 双方同亮 10 分钟，主屏显示「共时 10 分钟」。
> 点对方灯三下、长按一下，对方 App 在前台则收到相同节奏闪振。
> 对方未在看时，信号写入记录；打开 App 后灯自动回放最新未读序列。
> 上滑切到「信号」页，可见 `· · · —` 形态记录，点击可回放。
> 长按自己灯熄灯；发送方看不到对方是否已看。

## 11. 技术概要

| 层 | 选型 |
|----|------|
| 客户端 | Swift + SwiftUI |
| 灵动岛 | ActivityKit + Live Activities |
| 叩击同步 | WebSocket 事件流 `{ type: tap \| hold, ts }` |
| 在线同步 | WebSocket 心跳（~30s） |
| 共时存储 | SwiftData + 云端永久同步 |
| 信号存储 | SwiftData + 云端永久同步（仅序列形态，无已读同步） |
| 后端 | Firebase / Supabase |

**叩击协议**：发送方按 `ts` 上传叩击事件；接收方落库为 `SignalSegment { pattern: [.dot/.dash], receivedAt, seen: Bool }`；间隔 > 2.5s 切段。

**回放**：本地按 `pattern` + 固定节拍的时序驱动灯闪与 Haptic，与实时送达一致。

**已看状态**：仅接收方本地 `seen = true`，不上传云端，发送方不可见。

**项目路径**：`acc/Accomi/` · Bundle ID：`com.ccKu.Accomi`
